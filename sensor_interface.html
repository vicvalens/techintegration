Multiplatform workflow

The same ESP32 data source can feed different interfaces.

1

Sensor

Reads pressure, flexion, EMG, GSR, wheel rotation, or tilt.

2

ESP32

Sends Serial CSV and serves JSON through Wi-Fi.

3

Flet

Reads Serial CSV through USB and saves CSV.

4

Web / App

Reads JSON from /data using fetch.

5

Interface

Shows value, state, graph, and interpretation.

Live sensor trend

The graph shows the last values received from the ESP32 as an auto-scaled clinical trend line. If the ESP32 is not connected, it can run in simulation mode.

Future AI layer

The webpage can describe what AI could do later if enough sensor data is collected.

Classify

Identify user states such as safe movement, excessive effort, high pressure, or unstable posture.

Predict

Estimate fatigue, risk moments, or changes in activity based on previous sessions.

Recommend

Suggest rest, adjustment, assistance level, or feedback changes according to the indicator.
